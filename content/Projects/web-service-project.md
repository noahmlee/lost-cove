## Web service middleware project
While working in QA with a client that creates custom applications for shipping companies, there was need of a backend-only web-service to integrate customer databases to the app front end.

### Requirements
I was needed to build a web server hosted on the public internet, that can receive api requests from the app and return data from the user's database. To do this, I needed to expose API endpoints for authentication, and shipments. The project must also be connected to an actual dynamic database, no hard-coded JSON allowed.
